# UI Reference Notes

> `design/ui/` 아래 파일들은 실제 제품 UI의 정답이 아닙니다.

## 1. 사용 원칙

- `design/ui/`의 HTML과 스크린샷은 앞으로 실제 UI 디자인을 진행할 때 레이아웃 구조를 참고하기 위한 자료입니다.
- 현재 기획서와 맞지 않는 텍스트, 정책, 버튼, 상태, 색상, 화면 구성이 포함될 수 있습니다.
- 이 UI 디자인을 맹신해서는 절대 안 됩니다.
- 실제 UI를 설계하거나 구현할 때는 `design/design.md`, `design/architecture.md`, `design/api-spec.md`, `design/erd.md`의 최신 정책을 우선합니다.
- `design/ui/core_precision/DESIGN.md`의 색상, 타이포그래피, 간격 규칙도 참고용입니다. 최종 디자인 시스템은 실제 앱 구현 과정에서 다시 정리합니다.
- `design/ui/` 화면은 레이아웃 배치, 정보 밀도, 사이드바/상단바/패널 구조를 빠르게 파악하기 위한 레퍼런스로만 사용합니다.

## 2. 화면 번호 매핑

`design/ui/_1`부터 `design/ui/_14`까지는 다음 화면을 의미합니다.

이 매핑 표도 참고용입니다. `_1`, `_2` 같은 번호와 아래 의미를 고정된 정답으로 맹신하면 안 됩니다. 실제 UI를 구현할 때마다 최신 기획, 현재 구현 상태, 사용자 흐름, 화면 목적을 다시 확인하고 그 시점의 최선의 레이아웃을 선택합니다.

| 경로 | 의미 | 참고 포인트 | 주의할 점 |
| --- | --- | --- | --- |
| `design/ui/_1` | 과제 상세 화면 | 문제 링크, 제출 상태, 그룹원 제출 현황, 제출 버튼 배치 참고 | `Pass`, 채점처럼 현재 기획에 없는 표현은 사용하지 않습니다. |
| `design/ui/_2` | 과제 생성 화면 | 사이드바가 있는 과제 생성 폼, 제목/URL/deadline/정책 설정 폼 배치 참고 | 과제 설명은 plain text이고, 실제 정책 항목은 최신 기획 기준으로 재구성합니다. |
| `design/ui/_3` | 캘린더와 제출 활동 화면 | 월간 캘린더, 제출 활동 기록, 제출 지표 배치 참고 | 개인 성과 대시보드처럼 보이는 요소는 그룹 캘린더 중심 기획에 맞게 조정합니다. |

| `design/ui/_5` | 코드 제출 화면 1안 | 과제 요약, 코드 붙여넣기 영역, 제출 이름 입력, 제출 전 확인 영역 참고 | “총 3번 제출”, “최종 성적” 같은 문구는 현재 기획과 맞지 않습니다. |
| `design/ui/_6` | 공지와 커뮤니티 화면 | 고정 공지, 커뮤니티 글 목록, 게시판 통계 배치 참고 | 파일/이미지 첨부는 MVP에서 지원하지 않고, 카테고리는 우선 `자유` 하나로 시작합니다. |

## 3. 실제 UI 설계 시 우선순위

실제 UI를 새로 설계할 때 우선순위는 다음과 같습니다.

1. `design/design.md`의 제품 정책.
2. `design/api-spec.md`의 API와 상태 모델.
3. `design/erd.md`의 데이터 귀속 관계.
4. `design/architecture.md`의 FE/BE/worker/Supabase/Redis 구조.
5. `design/ui/`의 레이아웃 참고 자료.

## 4. 특히 다시 설계해야 하는 부분

- 라이트/다크 모드에서 모두 성립하는 색상 토큰.
- 한국어/영어 전환 시에도 깨지지 않는 텍스트 폭과 줄바꿈.
- 주간 캘린더 기본 뷰.
- 모바일 캘린더의 점 표시와 날짜 상세 목록.
- 제출 코드 버전별 리뷰 귀속 UI.
- 제출 상세 화면에서 AI 봇이 작성한 댓글·코드 리뷰가 사람이 단 것과 같은 자리에 섞여 보이는 UI(GitHub Bot 스타일).
- AI 코드 리뷰 본문에 포함된 `+`/`-` diff 코드 블록 렌더링.
- 그룹 생성 화면(`/groups/new`)의 7개 그룹 규칙 + 4개 가입 방식 토글 + 최대 인원수 + 그룹 설명 입력 영역.
- 그룹 상세에서 그룹 코드 노출과 `코드 갱신` 확인 모달(이전 링크 일괄 무효화 경고).
- 이메일 초대 발송 폼과 발송 후 상태 목록.
- 번역 보기 버튼과 번역 결과 + 토큰 차감 안내 표시.
- 마감 후 과제 단위 **집단 코드 비교 분석**(5.4.5). `translationLanguage=none`이면 미노출. 파일별 diff 중심 화면 + 긴 Markdown 리포트 1개. 데드 코드 정적 분석 기반 강조는 diff 판단과 무관함을 사용자에게 안내.
- AppShell 헤더의 `설정` 아이콘 버튼 단일 진입(테마/언어/내 정보 진입 통합).
- 과제 삭제와 그룹 삭제 확인 UI.
- 알림 클릭 대상이 삭제되었을 때의 `존재하지 않는 페이지입니다` 상태.
- diff 인라인 코드 리뷰 카드의 접기/펼치기 칩 UI(접힘 시 작성자 아바타 + 답글 `+N`).
- 다중 라인 범위 리뷰의 배경 음영(primary tint) 표시(세로 strip 없이 음영만으로 R{start}-R{end} 영역 강조).
- 댓글/리뷰 본문 fenced code block의 shiki 기반 syntax highlight 렌더링.

## 5. 절대 그대로 가져오면 안 되는 요소

- 문제 본문을 서비스 내부에 표시하는 UI.
- 채점, Pass, 성적, 랭킹, 점수화 표현.
- 파일 업로드 또는 이미지 첨부 UI.
- 개인 캘린더 중심 UI.
- 공개 그룹 탐색·외부 검색 UI.
- 비공개 그룹/공개 그룹 토글 표현.
- 일회성 초대 링크의 만료 시각 또는 사용 횟수 입력 UI.
- 피드백 공개 정책 매트릭스(역할 × 단위 × deadline 전후) UI.
- 표절 탐지 UI.
- 과도한 통계 카드 중심 대시보드.
