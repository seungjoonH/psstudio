# 과제 등록자 UI 컨텍스트 노트

- 2026-06-10: 요청은 과제를 누가 등록했는지 UI에 표시하고 필터에도 추가하는 것이다.
- 2026-06-10: 과제 엔티티와 프론트 타입에는 `createdByUserId`가 있지만 닉네임/프로필 이미지가 없어 사람이 읽을 UI를 만들려면 응답 계약에 등록자 표시 정보를 추가해야 한다.
- 2026-06-10: 과제 목록 카드에서는 기존 알고리즘 배지 줄을 침범하지 않고, 등록자를 별도 보조 메타 행으로 표시한다. 기존 정보 누락 없이 새 정보만 추가하는 것이 핵심이다.
- 2026-06-10: 필터는 기존 모달 필터 구조에 맞춰 등록자 섹션을 추가한다. 그룹별 목록은 그룹 멤버 중 실제 등록자를 옵션으로 쓰고, 전체 목록은 과제 응답에서 등록자를 모아 옵션을 만든다.
- 2026-06-10: 그룹 캘린더도 별도 과제 필터를 갖고 있어 등록자 필터를 같이 추가했다. 캘린더 일자 모달에는 목록 카드와 같은 등록자 메타를 작게 표시한다.
- 2026-06-10: 승인된 추가 범위에 맞춰 전체 캘린더 데이터 전달, 과제 상세 hero, 홈 해야 할 일 카드에도 등록자 표시를 추가한다. 제출 row, 알림, 브레드크럼, 설정 화면은 1차 범위에서 제외한다.
- 2026-06-10: 목록/홈/캘린더 모달의 작은 등록자 아바타도 대상자 아바타와 혼동될 수 있어 제거한다. 대신 대상자 수를 분모, 제출한 고유 사용자 수 전체를 분자로 하는 제출 진행도를 표시한다. 비대상자가 제출하면 `9/8`처럼 분자가 분모를 넘을 수 있으며, 막대 폭만 100%로 캡한다.
- 2026-06-10: 등록자 정보는 필터 옵션과 과제 상세 화면에서 유지한다. 컴팩트 카드와 캘린더 날짜 모달은 등록자보다 `내 제출 여부`, `전체 제출 규모`, `마감까지 남은 시간`을 우선한다.
- 2026-06-10: 캘린더 날짜 모달에는 해결/미해결 배지, `D-N`/마감 배지, 제출 진행도를 함께 둔다. 이 조합으로 내가 냈는지, 남들이 얼마나 냈는지, 마감까지 얼마나 남았는지를 한 카드 안에서 확인한다.
- 2026-06-10: 과제 목록 카드의 제출 진행도는 전체 폭을 쓰지 않고 오른쪽 상태 영역에 짧게 표시한다. 모바일에서는 진행도 폭을 더 줄여 제목·대상자·플랫폼·난이도 배지가 한 줄에서 과하게 밀리지 않도록 한다.
