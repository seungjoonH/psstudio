# 테마 규칙

## 규칙
- 색상/간격/타이포는 토큰으로 정의한다.
- 라이트/다크 모드는 테마 변수로 지원한다.
- 컴포넌트 스타일은 토큰 기반으로 작성한다.

## Do
- 앱 루트에서 테마를 토글하고 CSS 변수를 전파한다.

## Don't
- 기능 컴포넌트에 하드코딩 hex 값을 흩뿌리지 않는다.

## 예시
```css
:root { --bg: #ffffff; --fg: #111111; }
[data-theme="dark"] { --bg: #0f1115; --fg: #f2f2f2; }
```

## 경계
- 테마 계층은 토큰과 모드 스위치를 소유한다.
- 컴포넌트 계층은 변수를 소비한다.

## 테스트 범위
- 라이트/다크 대비를 검증한다.
- 토큰 회귀 스냅샷을 검증한다.
- 변경 완료 전 프론트 테스트 또는 최소 타입체크/린트를 반드시 수행한다.
