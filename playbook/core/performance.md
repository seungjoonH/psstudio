# 성능

## 규칙
- 실제 비용이 확인된 경우에만 최적화한다.
- 사소한 계산에 메모이제이션을 남용하지 않는다.

## Do
- 측정 후 최적화한다.

## Don't
- 선제적으로 캐시/메모이제이션을 넣지 않는다.

## Do 예시
```ts
const sum = a + b; // trivial 계산에는 useMemo 불필요
```

## Don't 예시
```ts
const sum = useMemo(() => a + b, [a, b]);
```

## 경계
- 성능 튜닝은 측정된 병목 지점에서만 수행한다.
- 비용이 미미하면 가독성을 우선한다.

## 테스트 범위
- 최적화 전후 지표를 비교한다.
- 최적화로 인한 동작 회귀를 검증한다.
